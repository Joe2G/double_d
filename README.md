# DoubleD - أداة التحميل الخارقة 🚀

**DoubleD** هو نظام متكامل للتحميل الذكي يتكون من جزئين أساسيين:
1. **برنامج مكتبي (Electron App):** يعمل كمحرك معالجة خلفي (Native Host) لتحميل الفيديوهات بجودات عالية.
2. **إضافة متصفح (Chrome Extension):** تعمل كـ "رادار" لالتقاط روابط الفيديوهات مباشرة من المتصفح وإرسالها للمحرك.

---

## 🏗️ هيكل المشروع

### 1. المحرك الرئيسي (Electron App)
* **اللغة:** Node.js / Electron.
* **الوظيفة:**
    * إدارة الأدوات الخارجية (`yt-dlp` للتحليل و `aria2c` للتحميل).
    * إنشاء `Local Server` للتواصل مع الاكستنشن.
    * تحميل التحديثات تلقائياً من GitHub.
    * إدارة المهام (Pause/Resume/Remove) عبر بروتوكول `Aria2 RPC`.
* **أهم الملفات:**
    * `main.js`: القلب النابض، يدير دورة حياة البرنامج ويدعم التواصل مع الاكستنشن.
    * `services/YtdlpController.js`: المسؤول عن "قنص" بيانات الفيديو (الجودات، الروابط، الأحجام).
    * `services/Aria2Controller.js`: "البلدوزر"، المسؤول عن تحميل الملفات بسرعة فائقة عبر 16 اتصال متوازي.

### 2. إضافة المتصفح (Chrome Extension)
* **الوظيفة:**
    * مراقبة الصفحات (MutationObserver) لاكتشاف وجود فيديو.
    * حقن واجهة (UI) في مواقع الفيديو (مثل YouTube) تظهر للمستخدم خيارات الجودة.
    * التواصل المباشر مع البرنامج المكتبي عبر `Native Messaging`.
* **أهم الملفات:**
    * `background.js`: جسر التواصل مع البرنامج الرئيسي و "الرادار" الذي يلتقط روابط الميديا.
    * `content.js`: المسؤول عن التفاعل مع صفحة الويب (حقن الأزرار وعرض المنيو).
    * `content.css`: التنسيقات البصرية للواجهة.

---

## ⚡ المميزات التكتيكية
* **سرعة جنونية:** استخدام `aria2c` بـ 16 اتصال للسيرفر الواحد.
* **دعم ذكي:** استخراج الجودات الحقيقية (بما فيها الصوت فقط كـ MP3) باستخدام `yt-dlp`.
* **استقرار:** إعادة اتصال تلقائي في حال انقطع الربط بين المتصفح والمحرك.
* **أمان:** تنظيف تلقائي لأسماء الملفات لتتوافق مع نظام ويندوز (إزالة الرموز الممنوعة).

---

## 🛠️ تقنيات مستخدمة
* **Electron:** لبناء واجهة سطح المكتب.
* **yt-dlp:** لتحليل وفحص مصادر الفيديو.
* **Aria2:** لتحميل الملفات بمسارات متوازية.
* **Chrome Native Messaging:** للربط بين المتصفح ونظام التشغيل.

---

## 💡 كيف يعمل النظام؟
1. يفتح المستخدم الموقع (مثل يوتيوب).
2. الـ `content.js` يكتشف وجود فيديو ويرسل إشارة للمحرك.
3. يقوم `YtdlpController` بتحليل الرابط وإرجاع الجودات المتاحة.
4. تظهر قائمة للمستخدم، عند الاختيار يقوم `background.js` بإرسال أمر التحميل للمحرك.
5. يبدأ `Aria2Controller` في سحب الملف وتخزينه في `Downloads/DoubleD`.
